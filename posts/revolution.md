# We need a new, modern software stack

I am very convinced that we need a completely new software stack from the ground up
C is more than half a century old -- why still use it?
The backbone of all modern software is old -- we need an overhaul.
I'm not saying that old software is inherentally bad, I'm just saying
the software needs to be constantly improved if not completely rewritten
with a more modern lense. Software is like metal, after a while it'll rust, and
you'll need to replace it with something new.
